SearchFile$ (function)

This function will recursively search the given drive or folder for the specified file, ie. it will start in the given place and walk through that place and all its subfolders to locate the file. If the file was found, then it is returned inclusive its full qualified absolute path.

SYNTAX:

filespec$ = SearchFile$ (startDir$, mustHave$, file$)

INPUTS:

startDir$ (STRING)

  • This is the place where to start the search, it can be any valid drive or folder specification. If this is empty, then the search starts in the current working folder.
  • Note that searching a whole drive or a folder with many files may take very long and will freeze your program for that time. For best performance there should be not more than appox. 1000 files in that place (counted recursively).

mustHave$ (STRING)

  • This may be either empty or any (even partial) path specification, which must appear in the final path. It's useful to target the right file, if you have several files of the same name scattered over different subfolders.

file$ (STRING)

  • The name of the file to search, any given path is ignored.

RESULT:

filespec$ (STRING)

  • On success this is the full specification of the found file, ie. inclusive its full qualified path (starting with drive letter).
  • If the result is empty, then either of the following happened:
    • The given startDir$ was invalid or not found.
    • The file was not found in that place inclusive its subfolders.
