LStrip$ & RStrip$ (function)

These functions will remove a range of defined chars from the left or right side of a string. The behavior is in that very similar to LTRIM$ and RTRIM$, but these functions here are not limited to remove spaces and do also provide some special modes for typical all day uses.

SYNTAX:

stripped$ = LStrip$ (mode%, arg$)
stripped$ = RStrip$ (mode%, arg$)

INPUTS:

mode% (INTEGER)

  • Any of the operation mode constants defined in the global CONSTs section in the dev_framework\GuiAppFrame.bi file (see also below).

arg$ (STRING)

  • This is the string, from which you wanna strip all the chars defined by the given mode.

RESULT:

stripped$ (STRING)

  • The given string with all defined chars before or after it removed, depending on the used function.

MODE CONSTANTS:

stmZERO%

  • remove zeros (ASCII: 0)

stmCTRL%

  • remove control chars (ASCII: 0-31, 127 according to iscntrl() function)

stmBLANK%

  • remove blank chars (ASCII: 9, 32 according to isblank() function)

stmWHITE%

  • remove white space chars (ASCII: 9-13, 32 according to isspace() function)

stmQUOTE%

  • remove quotation marks (ASCII: 34)

SPECIAL MODES:

stmFIXED%

  • Trim partially uninitialized fixed length strings (combines stmZERO% and stmBLANK%).
    • May be required if fresh fixed length strings are filled without using the regular assignment method (str = "data"), but eg. with MID$(str, x, y) = "data" instead.

stmTEXT%

  • Find the start & end of the real text content in a string (combines stmCTRL%, stmBLANK% and stmQUOTE%), hence even blank chars inside quotes get removed:
    • eg. " " " ABC " " " becomes just ABC after using LStrip$() and RStrip$() on it.

stmVALUE%

  • Prepare a number string as argument for the VAL(...) function, ie. LStrip$() is used to find the first minus (-), +, dot (.), & or digit (0-9) from the left side, while RStrip$() is used to find the first digit (0-9) from the right side:
    • eg. My account is $-123.45, need next paycheck. becomes just -123.45 after both functions are used on it.
  • The LStrip$() function also performs some simple checks:
    • 1.) & is accepted only if followed by B, H or O (any case)
    • 2.) +, - is accepted only if followed by a dot (.) or digit
    • 3.) the dot (.) is accepted only if followed by a digit
  • However, there is still the chance of misformatted numbers, especially if you don't know the source of the numbers (external input etc.).
