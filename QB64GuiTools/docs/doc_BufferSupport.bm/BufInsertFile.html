BufInsertFile (sub)

This subroutine will insert the entire contents of the given file into the specified buffer starting at its current buffer position.

The call will also recalculate all bookmarks to maintain its respective positions. The current buffer position is updated accordingly and is on the first byte behind the inserted data after the call.

SYNTAX:

BufInsertFile handle%, fileSpec$

INPUTS:

handle% (INTEGER)

  • This is the handle of any valid buffer, in which you want to insert the file data.

fileSpec$ (STRING)

  • The name of the file to insert. You may also add a path, if required.
  • Make sure the specified file exists to avoid file loading errors, those are not caught by this function. Also if inserting exceeds the 1GiB buffer limit, an error occurs (see Common-Info).

NOTES:

  • No matter which line endings mode is used in the buffer, the inserted file's data is not converted automatically into the respective format, as this would corrupt data in binary files.
  • If you know you have inserted a plain text file, but you are not sure about the origin (which OS) of the file, then you should call the appropriate conversion subroutine ConvBufToNativeEol(), ConvBufToWinEol() or ConvBufToLnxMacEol() right after inserting the file, to convert the file data into the buffer's line endings format.
