FileToBuf (function)

This function will load the entire contents of the given file into a new buffer and return its buffer handle. The current buffer position is on the first byte, the buffer will use the OS native line endings mode and its initial state after loading the file is "unchanged".

SYNTAX:

handle% = FileToBuf% (fileSpec$)

INPUTS:

fileSpec$ (STRING)

  • The name of the file you wanna load into the buffer. You may also add a path, if required.
  • Make sure the specified file exists and its size is <= 1GiB to avoid file loading errors, those are not caught by this function (see Common-Info).

RESULT:

handle% (INTEGER)

  • The handle of the new loaded buffer, which you must later pass to the various other buffer functions to designate this buffer for operation.
  • If no more buffers are available, then error SBE_NoMoreBuffers is returned (see Common-Info).

NOTES:

  • Although the buffer is in OS native line endings mode after loading, the file's data is not converted automatically into the respective format, as this would corrupt data in binary files.
  • If you know you work with plain text files, but you are not sure about the origin (which OS) of the files, then you should call the ConvBufToNativeEol() subroutine right after loading, to convert the file data into your OS's native line endings format.
  • In alternative you can also force the buffer to use the one or other line endings mode using the subroutines ConvBufToWinEol() or ConvBufToLnxMacEol().
