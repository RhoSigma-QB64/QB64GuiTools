BufToFile (sub)

This subroutine will save the current contents of the designated buffer into the specified file. An existing file with the same name will be overwritten.

The buffer itself remains unaffected regarding the current buffer position, set bookmarks, search criteria and active line endings mode. Only its state is set to "unchanged" after saving the file, hence you could easily setup an auto-save via a timer routine and after save continue working with the buffer as usual.

SYNTAX:

BufToFile handle%, fileSpec$

INPUTS:

handle% (INTEGER)

  • This is the handle of any valid buffer, which you wanna save into a file.

fileSpec$ (STRING)

  • The name of the file for saving. You may also add a path, if required.
  • Make sure any specified path exists to avoid file saving errors, those are not caught by this function (see Common-Info).

NOTES:

  • If you save a plain text buffer and want the line endings of the saved file to be suitable for a specific OS, then you may call the appropriate conversion subroutine ConvBufToNativeEol(), ConvBufToWinEol() or ConvBufToLnxMacEol() right before calling this function.
