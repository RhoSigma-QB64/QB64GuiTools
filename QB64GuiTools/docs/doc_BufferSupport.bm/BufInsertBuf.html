BufInsertBuf (sub)

This subroutine will insert the entire contents of the given source buffer into the specified destination buffer starting at the destination's current buffer position.

The call will also recalculate all bookmarks of the specified destination buffer to maintain its respective positions. Its current buffer position is updated accordingly and is on the first byte behind the inserted data after the call.

The source buffer itself remains completely unchanged and in same state regarding its current buffer position, bookmarks, search criteria and active line endings mode, hence you can simply go on working with the source buffer as usual.

SYNTAX:

BufInsertBuf handle%, fromHandle%

INPUTS:

handle% (INTEGER)

  • This is the handle of any valid buffer, in which you want to insert the data (destination buffer).

fromHandle% (INTEGER)

  • This is the handle of another valid buffer, from which you wanna take the data (source buffer).
  • If inserting exceeds the 1GiB buffer limit, an error occurs (see Common-Info).

NOTES:

  • No matter which line endings mode is used in the destination buffer, the inserted data is not converted automatically into the respective format, as this would corrupt binary data.
  • If you know you have inserted a plain text buffer, but you are not sure about the source buffer's line endings mode, then you should call the appropriate conversion subroutine ConvBufToNativeEol(), ConvBufToWinEol() or ConvBufToLnxMacEol() right after inserting, to convert the data into the destination buffer's line endings format.
