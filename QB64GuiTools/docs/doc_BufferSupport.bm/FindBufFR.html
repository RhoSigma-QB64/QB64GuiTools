FindBufFwd & FindBufRev (function)

These functions will search for the criteria, which were earlier defined for the specified buffer using the SetBufFind() function. Depending on the called function the search will be performed in forward or reverse direction but in both cases starting at the current buffer position. If the criteria is found, then the found position is returned and it will also automatically become the new current buffer position. If the criteria is otherwise not found, then zero is returned and the current buffer position remains unchanged. The found position remains valid until you write or delete data in the respective buffer.

For a sequential search to find all occurrences of the criteria in the buffer, you would simply call the same direction search function ever again until you get no more hits.

SYNTAX:

found& = FindBufFwd& (handle%, findID%, method%, treat%)
found& = FindBufRev& (handle%, findID%, method%, treat%)

INPUTS:

handle% (INTEGER)

  • This is the handle of any valid buffer, in which you wanna search the specified criteria.

findID% (INTEGER)

  • The ID number of the criteria to search. You got the ID from the respective SetBufFind() call.

method% (INTEGER)

  • This flag determines how to use the specified criteria, ie. how the search shall actually be performed. It must be one of the flags defined in the simplebuffer.bi file (see also below).

treat% (INTEGER)

  • This flag determines how to treat any lower/upper case differences in the search criteria and the searched buffer data. It must be one of the flags defined in the simplebuffer.bi file (see below).

RESULT:

found& (LONG)

  • This is either the found position on success, or zero (0), if no more occurrences of the specified search criteria were found.
  • If the return value is negative, then it is any of the error numbers defined in the simplebuffer.bi file (see Common-Info).

METHOD FLAGS:

SBF_FullData

  • Do a regular search, hence look for the search criteria as a whole unbroken string.

SBF_Delimiter

  • Make a delimiter search, hence the search criteria is not used as whole string, but as a list of delimiter chars to look for. The function returns as soon as any of the listed chars is found.

SBF_InvDelimiter

  • Make a inverse delimiter search. Same as above, but here the function returns as soon as any char is found, which is not listed as delimiter.

TREATMENT FLAGS:

SBF_AsWritten

  • Simply treat all criteria and buffer data as is, hence "Foo" <> "foo".

SBF_IgnoreCase

  • Use this flag to ignore the written case of all data, hence "Foo" = "foo".
